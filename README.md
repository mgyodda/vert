# KSK Vertikal Static Site

Статический сайт, подготовленный для деплоя на GitHub Pages.

## Что уже подготовлено

- Все внутренние ссылки переведены на относительные пути.
- CSS, JS, изображения и JSON работают без привязки к корню домена.
- Добавлен cache-bust для стилей и скриптов.
- Сайт можно публиковать как в корень домена, так и в GitHub Pages вида `https://username.github.io/repository-name/`.

## Локальная проверка

Из корня проекта:

```bash
python -m http.server 8080
```

Открыть:

```text
http://localhost:8080
```

## Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите в него весь текущий проект.
3. Запушьте в ветку `main`.
4. В репозитории откройте `Settings -> Pages`.
5. В `Source` выберите `GitHub Actions`.
6. После пуша GitHub сам соберёт и опубликует сайт по workflow из `.github/workflows/deploy-pages.yml`.

## Если нужен свой домен

1. Добавьте файл `CNAME` в корень проекта.
2. Впишите в него ваш домен, например:

```text
ksk-vertikal.ru
```

3. После этого можно вернуть и обновить `canonical`, `schema.org url` и другие SEO-поля под финальный домен.

## Изображения со старого сайта

Если нужно повторно скачать старые изображения:

```bash
python download_images.py
```

## Примечание

Для GitHub Pages добавлен файл `.nojekyll`, чтобы публикация шла как обычный статический сайт без лишней обработки.
